## Project Knowledge

**Documentation Hierarchy:**
```
/README.md                        → All users (250-300 lines)
/src/ExcelMcp.McpServer/README.md → .NET devs (80-100 lines)
/src/ExcelMcp.CLI/README.md       → CLI users
/vscode-extension/README.md       → VS Code users (100-120 lines)
/gh-pages/index.md                → Comprehensive reference (450-5000 lines)
/FEATURES.md                      → Feature overview
/CHANGELOG.md                     → Centralized changelog (Keep a Changelog format)
```

**Skills Architecture (single source of truth):**
- `skills/shared/*.md` → auto-copied to skill references AND auto-generated as MCP prompts
- Skill-based clients (VS Code, Cursor) read `skills/excel-*/references/`
- MCP-only clients (Claude Desktop) read auto-generated `[McpServerPrompt]` methods
- Build: `dotnet build -c Release` generates SKILL.md and MCP prompts

**gh-pages Local Documentation Pattern:**
- Uses Jekyll includes from `_includes/` (auto-generated by `build.sh`)
- All docs use LOCAL pages (`/features/`, `/installation/`), NOT external GitHub links
- Adding new pages requires: update `build.sh`, create page file, update `.gitignore`, update `_includes/README.md`

**Count Verification (CRITICAL):**
```powershell
# MCP Server: count tool files
# CLI: count command group folders
# Operations: count per-tool switch cases
# ⚠️ CLI has FEWER tools/operations than MCP Server!
```
